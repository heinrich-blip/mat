### 5. Problem Solving:

**Issues identified so far (preliminary, pending full exploration):**

1. **Root-level pollution**: ~150+ non-essential files (SQL, shell scripts, markdown docs, JSON dumps, comments.txt) cluttering the monorepo root. The instruction file only documents `PHASE_1_COMPLETION_SUMMARY.md`, `PHASE_2_QUICK_START.md`, and `REALTIME_USAGE.md` as expected root docs.

2. **Undocumented `src/services/` directory**: Not in the architecture spec. Contains a single file that likely belongs in `src/lib/` or `src/integrations/`.

3. **Undocumented `src/components/forms/` directory**: Not in the architecture spec. Needs investigation for what it contains and where files should go.

4. **32 loose files in `src/components/` root**: Major violation of the feature-based subdirectory organization. Files like `JobCard*.tsx` should be in `maintenance/`, `Tyre*.tsx` in `tyres/`, `Wialon*.tsx` in `wialon/`, `Vehicle*.tsx` in `Vehicle/`, `FuelBunkerManagement.tsx` in `diesel/`, `InventoryPanel.tsx` in a missing `inventory/` directory, etc.

5. **CSS files mixed with TSX**: `Dashboard.css` and `UnifiedMapView.css` sit in `src/components/` root alongside component files.

6. **Missing `inventory/` component subdirectory**: Documented in instructions but not present in actual directory listing (though `InventoryPanel.tsx` sits loose at components root).

7. **Naming inconsistency in hooks**: Mix of kebab-case (`use-toast.ts`) and camelCase (`useToast.ts`-style naming like `useDrivers.ts`).

8. **Potential duplicate Supabase types**: `src/types/supabase.ts` (130KB) alongside `src/integrations/supabase/types.ts` (the canonical location per instructions).

9. **`Vehicle/` directory uses PascalCase** while all other component subdirectories use lowercase — inconsistency acknowledged in instruction file but still flaggable.
